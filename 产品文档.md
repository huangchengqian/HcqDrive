# HcqDrive - 安卓局域网云盘

> 参考 **MiXplorer** 的功能深度,聚焦 **局域网单端服务** 场景。
> 安卓端是唯一服务端,客户端通过浏览器访问,无需安装任何 App。

---

## 1. 产品概述

### 1.1 一句话定义
HcqDrive 是一款运行在安卓设备上的 **局域网个人云盘服务**。开启后,同一 WiFi 下的任何设备(电脑/手机/平板)通过浏览器即可访问手机里的照片、视频、文档,支持浏览、上传、下载、管理。

### 1.2 产品定位
| 维度 | 定位 |
|---|---|
| 服务端 | **仅安卓**(Android 7.0 / API 24+) |
| 客户端 | **浏览器**(Web UI),无 App 安装 |
| 网络范围 | **仅局域网**(同 WiFi 或同热点) |
| 核心对标 | MiXplorer 的 Web Server + AirDrop 的易用性 |
| 商业模式 | 个人项目 / 开源(待定) |

### 1.3 设计原则
1. **零安装** — 客户端用浏览器,扫码即用
2. **隐私优先** — 数据不出局域网,不上传任何云端
3. **MiXplorer 级深度** — 文件管理功能参考 MiXplorer,做到专业级
4. **手机当服务器** — 续航友好,资源占用低
5. **优雅保活** — 借助前台服务 + 通知,做到长时间稳定运行
6. **精致 UI** — Web UI 和原生 UI 都要做到 **产品级精致**:
   - 统一的视觉语言(色彩 / 字体 / 间距 / 圆角 / 阴影)
   - 流畅的微交互动效(过渡、反馈、加载状态)
   - 暗色 / 浅色双主题,完整适配
   - 不容忍"占位 UI"和"半成品视觉"
   - 安卓原生配置页同样遵循 Material You + Compose 设计规范

## 2. 用户场景

### 2.1 核心场景

**S1: 拍完照,在电脑上修图**
- 摄影师用安卓手机拍完照,想在大屏电脑上挑图、修图
- 手机开 HcqDrive,电脑浏览器输入 `http://手机IP:8080`,看到所有照片,下载到电脑

**S2: 手机存满了,清理文件**
- 微信下载/抖音缓存占满手机,想批量导出
- 电脑浏览器进入,选中 N 个文件,一键下载 ZIP 压缩包

**S3: 朋友间传大文件**
- 给朋友传 2GB 的视频,微信发不了
- 双方连同一 WiFi,朋友扫码访问,直接下载

**S4: 备份新拍的照片到电脑**
- 出差回来,把当天所有照片传到工作电脑
- 增量浏览,按日期筛选,批量下载

**S5: 远程投屏照片给家人**
- 父母手机里有一堆老照片
- 投到客厅电脑大屏,全家一起看

### 2.2 用户画像
- **主要**: 拥有安卓旗舰/中端机、注重隐私、有文件管理需求的技术用户
- **次要**: 经常需要手机电脑互传文件的普通用户
- **设备组合**: 安卓手机 + Mac/Win/Linux 电脑(均通过浏览器访问)

## 3. 功能模块

### 3.1 服务端(安卓 App)

#### 3.1.1 服务控制
- [ ] 启动 / 停止 Web 服务
- [ ] 自定义端口(默认 8080)
- [ ] 启动时自动开启(可配置)
- [ ] 前台服务 + 常驻通知栏

  - 通知快捷操作:停止服务、复制地址、生成二维码
- [ ] 省电策略切换:性能优先 / 续航优先(限制并发)

#### 3.1.2 共享根目录
- [ ] **预设目录**(只读,系统相册):
  - `Pictures/`
  - `DCIM/`(相机照片)
  - `Movies/`、`Videos/`
  - `Music/`、`Audio/`
  - `Download/`
- [ ] **用户目录**(完全读写):
  - 内部存储根目录(选择性开放)
  - 外部 SD 卡(如有)
- [ ] 单个目录可设置:只读 / 读写 / 隐藏
- [ ] 目录可重命名(在 Web UI 里的别名)

#### 3.1.3 权限与鉴权
- [ ] **首次配对**:启动时生成 6 位数字配对码,有效期 5 分钟
- [ ] **配对方式**:
  - 浏览器输入配对码
  - 扫描二维码(自动填入)
- [ ] **会话保持**:配对成功后,设备 Token 存浏览器,30 天免重新配对
- [ ] **可选密码**:为当前会话设置一个 6+ 位密码,所有访问都要输
- [ ] **白名单设备**:已配对设备列表,可单独撤销

#### 3.1.4 应用设置
- [ ] 端口、协议(HTTP/HTTPS)
- [ ] 服务名称(mDNS 主机名,默认 `hcqdrive.local`)
- [ ] 是否允许上传
- [ ] 是否允许删除
- [ ] 是否显示隐藏文件
- [ ] 主题色(深色 / 浅色 / 跟随系统)
- [ ] 日志查看(最近 100 条访问记录)
- [ ] 电池优化白名单引导

### 3.2 客户端(Web UI)

#### 3.2.0 设计规范(M1 必交付)
- **响应式断点**(参考现代 SaaS 习惯):
  - 手机竖屏: `< 640px`  — 单列,底部操作栏
  - 手机横屏 / 小平板: `640px - 1024px`  — 列表为主,可隐藏侧栏
  - 平板 / 桌面: `≥ 1024px`  — 三栏布局(侧栏 + 主区 + 详情/预览抽屉)
  - 宽屏桌面: `≥ 1440px`  — 三栏 + 更大的文件网格列数
- **设计 Token**(统一变量,便于换肤):
  - 主色 + 4 阶饱和度梯度(50/100/500/700/900)
  - 中性色: 9 阶灰度
  - 语义色: success / warning / danger / info 各 2 阶
  - 间距: 4 的倍数(4 / 8 / 12 / 16 / 24 / 32 / 48)
  - 圆角: `sm=4` / `md=8` / `lg=12` / `xl=16` / `full=9999`
  - 阴影: 3 阶(无 / card / floating)
  - 字体: 系统字体栈 + 中英文混排优化,正文 14/15px
  - 动效: `ease-out` 200ms 为主,关键交互 300ms
- **图标库**: Lucide Icons(线条统一、可缩放、品类全)
- **空白**是设计语言,不要塞满
- **不交付** 半成品视觉;所有按钮、菜单、空状态、错误页都要有真实设计

## 4. 非功能需求

### 4.1 性能
| 指标 | 目标 |
|---|---|
| 首页加载(1000 文件) | < 1s |
| 缩略图生成(单张) | < 200ms |
| 局域网传输速度 | 跑满 WiFi 协商速度(实测 50-200 MB/s) |
| 大文件下载 | 支持断点续传,无大小上限 |
| 服务启动时间 | < 2s |
| 内存占用(空闲) | < 100MB |
| 并发客户端 | 默认 10,可选 50 |

### 4.2 安全
- **网络隔离**:仅监听局域网接口,绑定 `0.0.0.0` 时禁止外网访问
- **HTTPS 可选**:支持自签证书,避免局域网嗅探
- **配对码**:6 位数字,5 分钟过期
- **Token**:32 位随机字符串,HttpOnly Cookie 存储
- **路径遍历防护**:服务端严格校验,防止 `../` 越权
- **MIME 嗅探防护**:Content-Type 强制设置,`X-Content-Type-Options: nosniff`
- **日志脱敏**:访问日志不记录 Token 和 Cookie

### 4.3 兼容性
- **Android 版本**:minSdk 24 (Android 7.0),targetSdk 34 (Android 14)
- **处理器架构**:arm64-v8a(主流),armeabi-v7a(老设备),x86_64(模拟器)
- **存储**:内置存储 + 外部 SD 卡(API 24+ 需要 SAF)
- **浏览器**:Chrome 90+ / Safari 14+ / Firefox 88+ / Edge 90+
- **Web 标准**:ES2020 + 原生 API,无框架依赖(可选 Vue 3 构建)

### 4.4 可靠性
- **服务异常自恢复**:监听端口失败时尝试 +1 端口,最多 3 次
- **优雅关闭**:收到关闭信号时,完成所有进行中的下载/上传
- **大文件不丢**:传输中断时,临时文件清理机制
- **配置持久化**:所有设置存 DataStore,卸载前不丢失

### 4.5 可维护性
- **日志分级**:DEBUG / INFO / WARN / ERROR,可在设置中切换
- **崩溃上报**(可选,默认关闭):匿名堆栈
- **功能开关**:核心功能有运行时开关,方便调试

### 5.4 API 设计(RESTful)

```
认证
  POST   /api/auth/pair          { code } → { token }
  POST   /api/auth/verify        { token } → { valid }

文件系统
  GET    /api/fs/list?path=...   → { entries: [...] }
  GET    /api/fs/stat?path=...   → { meta }
  POST   /api/fs/mkdir           { path, name }
  POST   /api/fs/rename          { path, newName }
  POST   /api/fs/move            { src, dst }
  POST   /api/fs/copy            { src, dst }
  POST   /api/fs/delete          { path }
  GET    /api/fs/search?q=...

文件传输
  GET    /api/file/raw?path=...           (流式下载,支持 Range)
  GET    /api/file/zip?paths=...          (打包下载)
  POST   /api/file/upload                 (分片上传)
  GET    /api/file/thumb?path=...&size=   (缩略图)

媒体
  GET    /api/media/photos?year=...&month=...
  GET    /api/media/exif?path=...

分享
  POST   /api/share/create       { path, ttl, maxDownloads, password }
  GET    /api/share/info?token=
  DELETE /api/share/:token

系统
  GET    /api/status             { uptime, connections, ... }
  GET    /api/logs?limit=100
```

### 5.5 协议说明
- **传输协议**:HTTP/1.1 + WebSocket
- **编码**:UTF-8
- **数据格式**:JSON(API)、multipart/form-data(上传)
- **鉴权**:Cookie(`auth_token`) + 配对码
- **实时通信**:WebSocket 用于上传进度、目录变更推送

## 8. 风险与决策项

### 8.1 已确认决策
1. **应用包名**:`com.hcqdrive.app`
2. **应用名称**:`HcqDrive`
3. **开源**:是(具体 LICENSE 在 M5 选定,默认 MIT 或 Apache-2.0)
4. **付费 / 广告**:无,纯免费 + 开源
5. **账号系统**:无,局域网场景无意义
6. **客户端形态**:**Web UI 一套,主动适配手机 / 平板 / 桌面三端**(响应式)
7. **UI 精致度**:**M1 必交付项**,详见 3.2.0 设计规范
8. **桌面浏览器优先**:`≥ 1024px` 视为主要使用场景(配合用户"电脑访问"主诉求)
9. **平板支持**:`≥ 768px` 启用三栏布局,横竖屏都可用
10. **手机浏览器**:`< 640px` 兜底可用,但不是重点优化对象

### 8.2 已知风险
- **厂商后台限制**:华为/小米/OPPO 需手动加白名单,需在 App 内做引导页
- **Scoped Storage 兼容**:API 29+ 需通过 SAF 访问任意目录
- **大文件下载**:浏览器对单个连接有限制,需正确处理 HTTP Range
- **mDNS 不稳定**:部分路由器会屏蔽 mDNS 多播,需提供 IP 访问兜底
